## 2025-05-15 - [Secure Administrative Override in setUserState]
**Vulnerability:** The `setUserState` endpoint in `src/api/routes/users.ts` previously only allowed users to update their own state and strictly filtered updates through a whitelist, preventing administrators from modifying sensitive exam-related fields for any user.
**Learning:** While whitelisting is a strong security pattern for user-submitted data, it can inadvertently block legitimate administrative workflows. Implementing an authenticated administrative override (e.g., for `UserManager`s) ensures that privileged users can manage the system effectively while keeping regular users restricted.
**Prevention:** When implementing field whitelists or self-update restrictions, always consider whether an administrative override is required for that functionality. Use a clear role-based access control (RBAC) check (e.g., `isPermitted(me.roles, 'UserManager')`) to grant additional privileges to specific users.
