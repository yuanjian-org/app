## 2025-05-14 - Open Redirect and javascript: XSS via Callback URLs
**Vulnerability:** Insecure usage of user-provided `callbackUrl` in query parameters for frontend redirects after login or profile setup. Attackers could redirect users to malicious domains (Open Redirect) or execute arbitrary scripts via the `javascript:` protocol (XSS).
**Learning:** Next.js `router.push()` and `window.location.href` do not automatically sanitize URLs, making them vulnerable if they accept unsanitized query parameters. Relying solely on `startsWith('/')` is insufficient as `//example.com` is a valid protocol-relative URL that can lead to external domains, and `javascript:alert(1)` could be bypassed if not strictly checked.
**Prevention:** Use a centralized utility like `getSafeCallbackUrl` to enforce that redirects only occur to safe, relative paths starting with a single `/` and explicitly rejecting `//` and `/\`. This should be applied to all frontend redirection logic that handles user-provided paths.
